# 开源心理测评资源研究与《深宫命格》可借鉴方案

## 1. 结论

《深宫命格》不应该用随机分流算法，而应该采用“公开人格模型 + 宫廷情境题目 + 自有命格映射”的结构。

当前最值得借鉴的资源有四类：

1. IPIP：公共领域人格题库和计分键。
2. Big Five / HEXACO：成熟人格维度框架。
3. Forced-choice：相对选择式作答，降低用户选择“看起来更好”的答案。
4. Psychometrics / IRT 工具：有真实样本数据后，用于题目校准。

当前阶段不建议把复杂心理测量库直接放进前端。前端仍然保持轻量计分。真正要做的是用这些资源指导题库、维度和命格映射设计。

---

## 2. IPIP：最重要的公开题库参考

IPIP，全称 International Personality Item Pool，是公共领域的人格测评题库。它的价值不在于直接复制题目，而在于学习成熟量表如何构造题项、反向题、维度计分和多题交叉验证。

可借鉴点：

- 每个心理维度必须由多道题共同测量，不能只靠一两个剧情判断。
- 同一维度要在不同情境中重复出现，避免单一剧情误导结果。
- 要有正向题和反向题，例如“主动表达边界”和“为了关系牺牲边界”应共同测量分寸。
- 题项应尽量避免道德暗示，不让用户一眼看出“哪个选项更好”。

对《深宫命格》的转译方式：

| IPIP 常见题项逻辑 | 宫斗情境转译 |
|---|---|
| 容易担心事情会出错 | 贵妃一个眼神没有回应，你如何理解？ |
| 喜欢主动组织事情 | 宫中临时缺人统筹，你是否上前？ |
| 容易相信别人 | 新受宠的人主动示好，你如何接近？ |
| 会坚持自己的底线 | 同屋求你替她受过，你是否答应？ |
| 愿意影响别人决定 | 阵营冲突中，你是否主动改变局面？ |

## 3. Big Five：基础人格骨架

Big Five 常见五维是：开放性、尽责性、外向性、宜人性、神经质。它不是“类型论”，而是连续维度模型。用户不是某一种固定人，而是在多个维度上有不同高低。

对《深宫命格》的借鉴：

| Big Five 维度 | 本项目可转译维度 |
|---|---|
| 尽责性 / 自我控制 | 掌局、分寸 |
| 宜人性 / 合作与共情 | 人心 |
| 神经质 / 情绪敏感 | 心潮 |
| 外向性 / 社交主动性 | 掌局、人心 |
| 开放性 / 探索与灵活性 | 破局、权变，可作为后续扩展维度 |

Big Five 不足以完整解释宫斗语境里的权力动机，所以还需要参考 HEXACO。

---

## 4. HEXACO：补足权力与资源动机

HEXACO 在 Big Five 的基础上加入 Honesty-Humility，即诚实-谦逊维度。这个维度对《深宫命格》很重要，因为宫廷情境中大量选择都涉及地位、资源、交换、特权和权力距离。

本项目中的“权欲”维度，不能简单理解成坏。它更接近：

- 是否重视位置与资源。
- 是否愿意进行策略性交换。
- 是否会主动借势。
- 是否认为没有位置就没有解释权。
- 是否愿意用影响力保护自己。

表达上要避免道德审判。高权欲不是“坏”，低权欲也不是“好”。高权欲可以是现实、清醒、进取；低权欲可以是抽离、不恋权。

## 5. Forced-choice：更适合女性向 H5 的题型

传统同意度题很容易让用户选择看起来更体面、更善良、更成熟的答案。Forced-choice 强迫选择题更适合《深宫命格》：每个选项都合理，但指向不同心理偏好。

推荐题型：

> 宫宴上，你的位置被人占了。你更可能：
>
> A. 先找合适位置坐下，观察谁在看你。
> B. 微笑回应，宴后找负责人确认。
> C. 当场请人核对座次，不让事情含混。
> D. 顺势坐到更显眼处，把尴尬变成机会。

四个选项没有绝对好坏：

- A 测藏锋、风险规避。
- B 测关系调节、分寸。
- C 测边界、直接表达。
- D 测掌局、权力动机、机会感。

## 6. IRT / psychometrics 工具：有数据后再用

py-irt 是 Python 的 IRT 库，用 Pyro 和 PyTorch 拟合贝叶斯项目反应理论模型。它适合在有大量答题数据后分析题目质量，例如题目区分度、潜在特质估计、题目是否有效。

当前不建议直接接入前端。原因：

- 现在还没有真实样本数据。
- H5 前端不适合跑复杂统计模型。
- IRT 的价值在于“数据后校准”，不是“上线前凭空生成”。

R 的 psych、mirt、lavaan 等包适合做：

- Cronbach alpha 内部一致性。
- 探索性因素分析 EFA。
- 验证性因素分析 CFA。
- 多维 IRT。
- 题项区分度分析。

这些可以作为后期离线分析工具，不必进入前端代码。

## 8. 对当前代码的建议

不建议：

- 不建议直接复制公开量表原题到 H5。
- 不建议声称专业测评。
- 不建议用单一最高分决定命格。
- 不建议为了分散结果硬调算法。

建议：

- 保持六维人格底座。
- 每个维度至少由 8-10 个选项贡献。
- 每道题同时测 2-3 个维度，而不是只测一个维度。
- 每个命格用六维 profile 表示。
- 结果由 profile 匹配得出。
- 建立校准样本集，持续调整题目权重和 profile。
