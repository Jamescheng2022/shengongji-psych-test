# 《深宫命格》心理测评校准计划

## 1. 为什么需要校准

当前版本已经从“宫斗抽签”升级为“六维心理剖面 + 命格 profile 匹配”。但这还不是严格心理量表。真正可靠的测评需要通过大量作答数据校准题目、权重和结果分布。

本计划的目标不是让每个命格平均出现，而是避免以下问题：

- 某个中间命格吸走大多数用户。
- 全选 A / B / C / D 得到高度相似结果。
- 选项看似不同，但底层权重过于接近。
- 某些维度长期偏高或偏低。
- 结果解释和关键选择无法对应用户真实作答。

核心原则：**结果分布要自然，不要机械平均；结果解释要可信，不要为了分散而分散。**

---

## 2. 当前六维模型

| 维度 | 含义 | 高分倾向 |
|---|---|---|
| 掌局 agency | 主动性、控制感、目标驱动 | 主动改变局面，争取资源 |
| 藏锋 caution | 谨慎、防御、风险规避 | 观察后行动，保留退路 |
| 人心 affiliation | 共情、关系连接 | 重视情义和他人感受 |
| 分寸 boundary | 边界、自我保护 | 能拒绝，不轻易被吞没 |
| 心潮 emotionality | 情绪敏感、压力反应 | 对评价和关系变化敏感 |
| 权欲 power | 地位、资源、影响力动机 | 善借势，重视位置和筹码 |

---

## 3. 校准测试集

上线前必须至少跑以下固定路径。每条路径要记录：

- 最终命格。
- 六维分数排序。
- 命格解释是否符合路径意图。
- 是否被“玲珑女官”等中间型吸走。

### 3.1 机械路径

| 路径 | 目的 | 合理预期 |
|---|---|---|
| 全选 A | 检查 A 是否长期偏稳、偏边界 | 不应总是同一命格，若同一命格需解释合理 |
| 全选 B | 检查 B 是否偏关系/共情/缓冲 | 可能偏温仪贵人、玲珑女官或藏锋类 |
| 全选 C | 检查 C 是否偏观察/试探/分寸 | 可能偏玲珑女官、藏锋美人、清醒旁观者 |
| 全选 D | 检查 D 是否偏权力/破局/主动 | 可能偏孤傲凤仪、笑面权臣、智谋贵妃 |

注意：机械路径不是正常用户行为，不要求结果“好看”，但要用来检测算法偏置。

### 3.2 人格路径

| 路径名称 | 作答模式 | 合理预期 |
|---|---|---|
| 共情型 | 多选照顾别人、承受关系压力、害怕伤害他人 | 温仪贵人 |
| 边界型 | 多选拒绝、守底线、只承担自己的责任 | 孤傲凤仪 / 藏锋美人 / 清醒旁观者 |
| 权力型 | 多选借势、交换、押注、上升 | 笑面权臣 / 冷月谋后 / 智谋贵妃 |
| 藏锋型 | 多选观察、忍耐、记录、等待 | 藏锋美人 / 冷月谋后 / 清醒旁观者 |
| 控局型 | 多选组织、复盘、调整局势 | 智谋贵妃 / 冷月谋后 |
| 清醒抽离型 | 多选退开、自守、少卷入 | 清醒旁观者 |
| 孤勇型 | 多选当场表达、正面承担、为自我或他人站出来 | 孤傲凤仪 / 温仪贵人 |
| 玲珑周旋型 | 多选缓冲关系、试探、互惠、保留余地 | 玲珑女官 |

## 4. 结果分布预期

真实用户分布不应机械平均，但也不应过度集中。

在没有真实数据前，内部测试的暂定目标：

| 命格 | 合理占比范围 |
|---|---|
| 智谋贵妃 | 8%-18% |
| 温仪贵人 | 8%-18% |
| 冷月谋后 | 6%-16% |
| 玲珑女官 | 10%-22% |
| 孤傲凤仪 | 6%-16% |
| 藏锋美人 | 8%-18% |
| 笑面权臣 | 6%-16% |
| 清醒旁观者 | 8%-18% |

如果某一命格在随机模拟中超过 30%，说明 profile 或算法可能存在吸附问题。

## 5. 题目质量检查

每道题需要检查四件事：

### 5.1 选项是否都合理

坏例子：

- A 是成熟理性。
- B 是善良。
- C 是愚蠢。
- D 是恶毒。

这种题会诱导用户选择社会赞许答案。

好例子：

- A 稳妥。
- B 重关系。
- C 守边界。
- D 抓机会。

四个都合理，只是人格偏好不同。

### 5.2 选项权重是否过度集中

每个选项最多应该强影响 2-3 个维度，不要同时给所有维度大幅加减。

建议范围：

- 主维度：+2 或 +3。
- 次维度：+1 或 +2。
- 反向维度：-1 或 -2。
- 不相关维度：0。

### 5.3 每个维度是否被足够测量

30 道题中，每个维度至少应被 12 个以上选项明显影响。否则该维度不稳定。

### 5.4 是否有反向测量

例如“分寸”不只通过拒绝测，也要通过“为了关系过度让步”反向测。

## 6. 算法校准原则

当前算法使用：

1. 累加六维权重。
2. 标准化为 0-100。
3. 计算用户人格形状与各命格 profile 的差异。
4. 同时考虑最高两个维度、最低一个维度。
5. 选择最接近的命格。

后续调试优先级：

1. 先调题目权重。
2. 再调命格 profile。
3. 最后才调算法惩罚项。

不要为了某个结果出现率低，直接硬编码结果。

## 7. 建议新增校准脚本

建议后续新增：

```text
scripts/calibrate-results.ts
```

脚本功能：

- 自动跑全 A / 全 B / 全 C / 全 D。
- 自动生成若干人格路径。
- 随机生成 1000 组答案。
- 输出每个命格出现次数。
- 输出每个维度平均分。
- 标出过度集中的命格。

示例输出：

```text
机械路径：
```

## 8. 后续真实数据采集建议

如果上线后有用户量，建议匿名记录：

- 每题选择的选项。
- 六维最终分数。
- 最终命格。
- 是否点击分享。
- 是否重测。
- 是否进入海报页。

不要记录姓名、手机号、微信 ID 等敏感身份信息。

这些数据用于：

- 检查题目区分度。
- 检查命格分布。
- 判断哪些结果更有分享欲。
- 发现用户在哪些题流失。

## 9. 版本路线

### V0.1 当前阶段

- 六维模型确定。
- 30 道宫斗题上线。
- 8 个命格 profile 匹配。
- 内部机械路径测试。

### V0.2 校准阶段

- 新增校准脚本。
- 调整权重和 profile。
- 修正过度集中结果。
- 优化关键选择解释。

### V0.3 数据阶段

- 增加匿名统计。
- 分析真实用户结果分布。
- 用数据修题。

### V1.0 稳定版本

- 保留稳定题库。
- 输出更精致的结果页和分享海报。
- 形成可复用测试模板。

---

## 10. 当前最重要的下一步

立即新增并运行校准脚本，验证：

1. 全 A / B / C / D 是否出现合理差异。
2. 偏共情、偏权力、偏边界、偏藏锋路径是否落到预期命格。
3. 随机模拟中是否还有某一命格超过 30%。
4. 结果解释是否能回溯到关键选择。
