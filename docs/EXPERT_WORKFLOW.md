# 专家分工与工作流

## 一、为什么要用三专家结构

这个产品不能只靠前端，也不能只靠文案。

它同时要求：

1. 看起来高级
2. 玩起来沉浸
3. 测起来可信
4. 结果说得准
5. 截图愿意发

所以必须用三专家协作，而不是单一角色。

## 二、三位专家的职责边界

### A. 中国古典审美前端设计师

核心目标：

> 让用户第一眼觉得这是高级、精致、女性向、值得完成的产品。

负责内容：

- 首页视觉
- 测试页阅读体验
- 结果页构图
- 海报页截图体验
- 动效节奏
- 音乐交互
- 移动端适配

判断标准：

- 是否像一个精心设计的产品，而不是套模板
- 是否一屏可读
- 是否有中国古典审美，但不土
- 是否留白足够
- 是否文字比装饰更重要

### B. 女性心理学家

核心目标：

> 保证每道题背后真的有心理构念，结果不是玄学糊弄。

负责内容：

- 测评维度
- 子维度定义
- 题目心理意图
- 选项权重
- 结果解释
- 风险边界

判断标准：

- 每道题能不能说明“测什么”
- 每个选项是否代表不同心理策略
- 是否避免明显正确答案
- 是否避免诊断化语言
- 结果是否既准又不伤人

### C. 宫斗剧本女编剧

核心目标：

> 让每一道心理题都像一幕真正的宫斗戏。

负责内容：

- 五章剧情推进
- 情境冲突
- 选项语言
- 人物关系
- 张力节奏
- 结果命格命名

判断标准：

- 情境是否有权力压迫
- 选择是否都有代价
- 选项是否像人会真的做出的反应
- 是否有女性关系的微妙感
- 是否避免幼稚化宫斗

## 三、题目生产流程

每道题经过 5 步：

### Step 1：心理学家定构念

先确定这题测什么。

例：

- 被公开质疑时的压力反应
- 面对资源诱惑时的边界感
- 被关系拉拢时的信任策略

### Step 2：编剧写场景

把抽象心理构念变成宫斗情境。

要求：

- 80-140 字
- 有人物、有压力、有后果
- 不解释心理学
- 不出现现代词

### Step 3：心理学家写选项权重

每个选项至少对应 2 个维度。

例：

```ts
weights: {
  agency: 2,
  defense: 1,
  relation: -1,
  risk: 0,
  boundary: 1
}
```

### Step 4：编剧润色选项

选项必须像剧情选择，不像问卷。

错误：

- “我会主动争取”
- “我会退让”

正确：

- “当场不争，只把那本错账册轻轻推到灯下”
- “先认下疏漏，夜里再让该看见的人看见”

### Step 5：前端设计师压缩展示

检查手机端是否可读。

标准：

- 场景不超过 140 字
- 选项每个不超过 32 字
- 4 个选项一屏可见
- 必要时将解释放入结果页，而不是题目页

## 四、结果生产流程

每个结果类型经过 4 步：

### Step 1：定义心理组合

例如：

- 高防御 + 高关系策略 + 中低风险
- 高边界 + 低关系依赖 + 中高主动性

### Step 2：命格命名

命名规则：

- 4-6 字
- 有古典感
- 不低俗
- 不像网游职业
- 不像星座标签

### Step 3：写双层解释

每个结果都要有两层：

- 深宫画像：你在故事里会如何活下去
- 心理画像：你在现实关系中如何保护自己

### Step 4：写传播金句

金句标准：

- 适合截图
- 用户愿意发
- 有被说中的感觉
- 不羞辱用户

## 五、旧项目迁移边界

旧仓库 `shengongji-h5-mvp` 可以参考，但不能整体复制。

### 可以复用

- Next.js 工程经验
- App Router 页面结构
- 音乐播放组件思路
- 本地存储思路
- 题库 JSON 驱动思路
- Vercel 部署经验

### 不建议复用

- 12 题结构
- 8 个结果类型限制
- 直接展示五维画像
- 复杂折叠面板
- 游戏化数值感
- “测一测你在后宫能活到第几集”这种偏娱乐标题

### 必须重做

- 30 题题库
- 12 个结果类型
- 专业心理测试说明
- 结果页结构
- 海报页构图
- 整体视觉语言

## 六、验收标准

### 产品验收

- 用户知道这是心理测试，不是小游戏
- 用户觉得 30 题值得做完
- 用户做题时不觉得像问卷
- 用户结果页能看到自己
- 用户愿意截图分享

### 专业验收

- 每题有明确测评意图
- 每个选项有合理权重
- 每个结果有心理解释
- 不宣称医学诊断
- 不使用版权受限正式量表原题

### 前端验收

- 手机端流畅
- 题目页不拥挤
- 选项易点
- 结果页层次清楚
- 海报页适合截图
- Vercel build 通过
