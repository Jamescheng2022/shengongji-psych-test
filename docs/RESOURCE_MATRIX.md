# 资源矩阵：哪些能现成改造，哪些必须自己造

## 一、可以现成改造的资源

### 1. 前端工程框架

可复用：

- Next.js App Router
- React 组件结构
- TypeScript 类型系统
- Tailwind CSS 或 CSS variables
- Vercel 自动部署

用途：

- 快速搭建 H5 页面
- 保证移动端性能
- 后续方便拆分题库、结果模型、海报组件

风险：

- 不要把现成 UI 模板直接套古风皮肤
- 不要依赖复杂动效库导致手机端卡顿

### 2. 心理学公开理论框架

可参考：

- Big Five / Five Factor Model：人格维度参考
- HEXACO：诚实谦逊、情绪性等维度参考
- Attachment theory：亲密关系和依恋反应参考
- Locus of control：控制感参考
- Coping styles：压力应对方式参考
- Dark Triad：权力、操控、自恋等边界参考

用途：

- 给测评维度提供心理学骨架
- 帮助结果页更专业
- 避免完全凭感觉写命格

限制：

- 不直接复制正式量表题目
- 不宣称临床诊断
- 不使用受版权限制的量表原题
- 不把娱乐测试包装成医学测试

### 3. 传统人格测试产品结构

可参考：

- 首页钩子
- 题目进度
- 选择动效
- 结果页结构
- 分享海报
- 复测入口

用途：

- 学习转化路径
- 学习传播机制
- 学习结果文案结构

限制：

- 不直接模仿 MBTI / 星座 / 塔罗文案
- 不做低级标签化
- 不做“你就是这种人”的绝对判断

### 4. 古典视觉素材思路

可参考：

- 宫墙色系
- 宣纸肌理
- 金线边框
- 印章构图
- 屏风层次
- 宫灯光影
- 宋元明清绘画留白感

用途：

- 建立视觉方向
- 生成 UI 纹理和装饰语言

限制：

- 不直接盗用影视剧剧照
- 不直接使用参考图作为背景
- 不使用有版权风险的宫斗剧图片
- 不让装饰压过文字阅读

## 二、必须自己造的资源

### 1. 测评模型

必须自研。

原因：

- 产品差异化来自“宫斗情境 × 心理构念”的映射
- 不能直接复制现成量表
- 需要为女性用户、压力场景、关系博弈重新设计

第一版建议 5 个一级维度：

1. 权力感 / Agency
2. 情绪防御 / Emotional Defense
3. 关系策略 / Relational Strategy
4. 风险偏好 / Risk Orientation
5. 自我边界 / Boundary & Selfhood

每个维度下设 2-3 个子维度。

### 2. 剧情题库

必须自研。

原因：

- 题目不是普通心理问题，而是宫斗场景里的行为选择
- 每道题都要同时满足：剧情合理、心理有效、女性视角真实、选择有张力

第一版建议：

- 30 道正式题
- 5 章，每章 6 题
- 每题 4 个选项
- 每个选项对应多个维度权重

### 3. 结果人格类型

必须自研。

原因：

- 结果是产品传播核心
- 必须既有古风命格感，又有心理解释
- 不能像普通游戏职业，也不能像星座糊弄

第一版建议 12 个结果类型：

- 冷宫复盘型
- 锦帐控局型
- 珠帘观人型
- 暗香避锋型
- 凤印执念型
- 宫墙自守型
- 玉阶试探型
- 青灯忍冬型
- 金钗破局型
- 镜湖藏锋型
- 长夜共情型
- 丹墀孤勇型

### 4. 结果解释文案

必须自研。

每个结果至少包含：

- 命格名
- 一句话判词
- 深宫画像
- 心理画像
- 关系模式
- 压力反应
- 隐藏风险
- 成长建议
- 分享金句

### 5. 视觉系统

必须自研。

原因：

- 这是用户第一眼判断高级感的地方
- 普通模板很容易变成土味古风

至少要自研：

- 色彩系统
- 字体层级
- 卡片样式
- 题目页布局
- 结果页构图
- 海报页构图
- 装饰纹理

## 三、整合原则

### 现成资源负责效率

- 工程框架
- 部署流程
- 页面路由
- 基础交互
- 心理学理论参考

### 自研资源负责壁垒

- 测评模型
- 剧情题库
- 结果类型
- 古典审美 UI
- 女性用户洞察

## 四、最小可上线版本

第一版不要追求完整心理学论文级严谨。

但必须做到：

1. 用户觉得画面高级
2. 用户觉得剧情像真的宫斗
3. 用户觉得题目是在测自己
4. 用户觉得结果说中了
5. 用户愿意截图发朋友圈
6. 用户愿意转给朋友测

## 五、绝对不要做

- 不要 12 题草草结束
- 不要纯随机命格
- 不要复杂数值面板
- 不要把心理学名词堆在结果页
- 不要把页面做成网游面板
- 不要把参考图直接当背景
- 不要使用影视剧版权图
- 不要宣称医学诊断
