# Test Page Implementation Plan

## 本轮目标

基于参考图，把 `/test` 从偏深红海报风调整为浅色宣纸宫廷风，形成真实可运行、可交互、可维护的 30 题移动端 H5 答题页。

## 保留的产品事实

- 保留正式 30 题，题目来源仍为 `lib/test-model/questions.ts`。
- 保留每题 4 个选项，界面显示为甲、乙、丙、丁。
- 保留当前六维模型：`agency`、`caution`、`affiliation`、`boundary`、`emotionality`、`power`。
- 不改题库内容。
- 不改 `computeResult` 计分逻辑。
- 不改 `saveAnswers`、`saveResult` 存储结构。
- 第 30 题完成后仍跳转 `/result`。

## 实现步骤

1. 确认当前分支为 `refactor/30-scene-test-page-v2`。
2. 审查 `/test` 当前单文件页面结构。
3. 确认活跃模型为 30 题、4 选项、六维评分。
4. 拆分测试页组件。
5. 新增 `app/test-page.css`，只作用于测试页 class 命名空间。
6. 将交互调整为“选择选项 -> 高亮 -> 点击下一题”。
7. 实现上一题返回、收藏 UI toggle、下一题保存与递进。
8. 复用原有 `saveAnswers`、`computeResult`、`saveResult`、`router.push("/result")`。
9. 做 375x667、390x844、430x932 截图检查。
10. 进行两轮视觉微调：先修底部遮挡，再处理长剧情卡片高度。
11. 验证从 `/test` 能完成到 `/result`。
12. 运行 `npm run build`。

## 视觉策略

- 背景：米白宣纸底，叠加轻微网纹、红金光晕。
- 顶部：圆形返回、居中标题、右侧胶囊占位按钮。
- 进度：红金渐变进度条，花形进度节点。
- 题卡：旧纸卡、细金边、四角装饰、朱砂花印。
- 选项：浅色圆角卡、甲乙丙丁朱砂印章、右侧箭头。
- 底部：上一题、收藏、下一题三段式操作区，下一题为红金主按钮。

## 风险控制

- 不新增依赖。
- 不修改首页、结果页、海报页。
- 不修改题库、计分、存储文件。
- 长剧情通过题卡内部轻滚动解决，不压住选项。
